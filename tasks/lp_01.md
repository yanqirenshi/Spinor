# 実装タスク: Spinor プロモーション用Webサイト

## 概要

`specs/24_website.md` で定義された仕様に基づき、Spinor の公式ランディングページを実装する。

## 担当者

*   **エンジニア:** Claude Code

## 前提条件

*   `specs/24_website.md` の内容を理解していること。
*   Git, HTML, CSS の基本的な知識があること。

## タスク一覧

### 1. 開発環境の準備

*   **変更ファイル:** なし
*   **具体的な変更内容:**
    *   プロジェクトのルートに `docs` ディレクトリを作成する。ここが Web サイトのソースコードを配置する場所となる。
    *   `docs` ディレクトリ内に、`index.html` と `style.css` という空のファイルを作成する。
*   **確認手順:**
    *   `ls docs` コマンドを実行し、`index.html` と `style.css` が存在することを確認する。

### 2. HTML 構造の実装

*   **変更ファイル:** `docs/index.html`
*   **具体的な変更内容:**
    *   `specs/24_website.md` の「4. ページ構成とコンテンツ」で定義された全てのセクション（ヘッダー, ヒーロー, 特徴, コード例, 始め方, フッター）に対応する HTML タグ（`<header>`, `<main>`, `<section>`, `<footer>`など）を記述する。
    *   各セクションには、適切な `id` 属性を付与してページ内リンクを可能にすること。
    *   `style.css` を読み込むための `<link>` タグを `<head>` 内に追加する。
    *   この時点では、詳細なコンテンツはプレースホルダーで良い。
*   **確認手順:**
    *   ブラウザで `docs/index.html` を開き、骨格となるセクションが正しく表示されることを確認する。

### 3. CSS スタイルの実装

*   **変更ファイル:** `docs/style.css`
*   **具体的な変更内容:**
    *   仕様書に基づき、ダークテーマを基調とした配色を定義する。
        *   例: `background-color: #1a1a1a;`, `color: #f0f0f0;`
    *   フォントサイズ、マージン、パディングなど、基本的なタイポグラフィとレイアウトを定義する。
    *   シンタックスハイライト用のスタイルを定義する（コード例セクションで使用）。
    *   `@media` クエリを使用して、モバイルデバイス向けのレスポンシブスタイルを記述する。
*   **確認手順:**
    *   `index.html` と連携させ、ブラウザで表示を確認しながらスタイルを調整する。

### 4. コンテンツの追加

*   **変更ファイル:** `docs/index.html`
*   **具体的な変更内容:**
    *   HTML の各セクションに、仕様書で定められたテキスト、キャッチコピー、コード例、コマンドなどを正確に記述する。
    *   GitHub リポジトリや `TODO.md` へのリンクが正しく設定されていることを確認する。（注意: GitHub のユーザー/リポジトリ名は実際の URL に置き換えること）
*   **確認手順:**
    *   ブラウザで最終的な表示を確認し、すべての情報が正しく、誤字脱字がないことを確認する。

### 5. ローカルでの動作確認

*   **変更ファイル:** なし
*   **具体的な変更内容:**
    *   以下のコマンドを実行し、ローカルサーバーを起動して表示を確認する方法を確立する。
        ```bash
        # プロジェクトのルートディレクトリで実行
        python -m http.server --directory docs 8000
        ```
*   **確認手順:**
    *   ブラウザで `http://localhost:8000` にアクセスし、作成した Web サイトが表示されることを確認する。

### 6. (オプション) GitHub Pages での公開

*   **変更ファイル:** なし
*   **具体的な変更内容:**
    *   リポジトリの "Settings" > "Pages" に移動する。
    *   "Build and deployment" の "Source" で "Deploy from a branch" を選択する。
    *   "Branch" で `main` (または `master`) ブランチと `/docs` フォルダを選択し、"Save" をクリックする。
*   **確認手順:**
    *   表示された `*.github.io` の URL にアクセスし、Web サイトが公開されていることを確認する。
