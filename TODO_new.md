# Spinor Project Roadmap (Issue-Driven Phase)

本ロードマップは、プロジェクトの成長に伴い再構築されたものです。
今後の開発は以下の7つの大枠（エピック）に分類され、GitHub Projects V2 と Issue をベースとしたハイブリッド・ワークフローによって自律的に管理・進行します。

## (1) Common Lisp Alignment (CL互換レイヤー)
既存の Lisp エコシステムとの親和性を高め、言語としての表現力と実用性を向上させる。

- [ ] **オペレータの充実:** Common Lisp の挙動にさらに近づけるための組み込み関数・マクロの継続的な追加。

## (2) コンパイラの進化とネイティブ基盤 (Compiler Evolution & Native Foundation)
LLVMへの移行やハードウェアAPIとの連携を通じて、実行性能を極限まで引き出す環境を構築する。

- [ ] **GCC依存からの脱却 (Step 36: LLVM Backend Investigation):**
    - `llvm-hs` ライブラリを用いた LLVM IR 生成の調査。
    - LLVM JIT でメモリ内コンパイル・実行するプロトタイプの作成。
- [ ] **ネイティブコンパイラの強化:** C言語トランスパイル方式の最適化、または LLVM バックエンドの統合。
- [ ] **科学技術計算と可視化基盤のネイティブ対応:**
    - **Step 47 (OpenCL):** Cトランスパイル時に OpenCL API を呼び出す Cコード生成の実装。
    - **Step 48 (OpenGL):** ネイティブビルド時の OpenGL/GLFW ライブラリのリンク対応。

## (3) ライブラリの充実 (Standard Library & Ecosystem)
言語が「実世界」とやり取りするためのインターフェースの構築。

- [ ] **HTTP / HTTPS リクエスト:** 外部ネットワークと通信するためのクライアント機能（`http-get`, `http-post` 等）の実装。
- [ ] **HTTP サーバー:** Spinor 上で稼働する軽量な Web サーバー機能の構築。
- [ ] **データベースアクセス:** SQLite や PostgreSQL などの DB と連携するためのライブラリ実装。

## (4) アプリケーションの構築 (Real-world Applications)
エコシステムの検証とドッグフーディングを兼ねた、実用的なソフトウェアの構築。

- [ ] **外部 Web API を利用したアプリ構築:**
    - (3) の HTTP ライブラリを活用した CLI ツールや、はてなブックマークAPI等を利用した実用アプリケーションの開発。
    - Windows 11向けの端末管理アプリケーションなどのプロトタイプ構築。
- [ ] **OpenGL を利用したアプリの構築:**
    - ネイティブ可視化基盤を利用した実用アプリ（※具体的なアプリのアイデアは今後の議論で決定する）。
- [ ] **OpenCL を利用したアプリの構築:**
    - GPGPUによる高速計算を利用した実用アプリ（※具体的なアプリのアイデアは今後の議論で決定する）。

## (5) 利用促進 (Promotion & Developer Experience)
開発者が Spinor を発見し、迷わず使い始めるための環境整備。

- [ ] **ドキュメントの充実:**
    - 全プリミティブ・標準ライブラリ関数を網羅したリファレンス (CLHS style) の完成。
    - GitHub Pages デプロイへの自動化と、ユーザーマニュアルの統合。
- [ ] **情報発信:**
    - **Step 58-B:** リポジトリ安定稼働後の、MELPA (`recipes/`) への公式パッケージ (`spinor-mode`) 登録申請。
- [ ] **サンプル・コード:** 実践的なユースケースを示す Cookbook やチュートリアルの継続的な追加。

## (6) テストの充実 (Testing & Quality Assurance)
言語コアやライブラリの信頼性を担保し、安全な開発サイクルを回すためのテスト基盤の強化。

- [ ] **テストフレームワーク:** Twister ライブラリ上で動作する、堅牢な標準テストツールの確立。
- [ ] **テストカバレッジの向上:** 言語コア（ASTパース・評価・型推論）および組み込み関数・標準ライブラリのエッジケースにおけるテスト拡充。
- [ ] **CI/CD パイプラインの高度化:** GitHub Actions でのテスト実行の最適化や、将来的なパフォーマンス回帰テストの導入検討。

## (7) 究極の目標: Lispマシンの創生 (The Ultimate Dream)
Spinor を単なるアプリケーションレベルの処理系から、ハードウェアと直接対話する「Lispマシン（OS / ハードウェア）」へと昇華させる。

- [ ] **The Ultimate Dream (Lispマシンの創生):**
    - **Step 51:** Lispマシンの歴史的背景とアーキテクチャの調査（ビジョンペーパー策定）。
    - **Step 52:** Linux 等に依存しない、Spinor OS / Unikernel (ベアメタル実行) のプロトタイプ作成。
    - **Step 53:** C/LLVM を経由せず、直接 FPGA/HDL (論理回路) にマッピングするハードウェア実行構想。
